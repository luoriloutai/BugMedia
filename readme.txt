# EGL和OpenGLES需要开启一个线程运行，并且要在这个开启的线程里初始化各种需要的对象，
  但是ANativeWindow并不需要在这个线程创建。

  遇到shader不能创建的问题困扰好久，原因就在于EGLSurface在线程外面创建了。OpenGL的东西都要
  在EGL绑定了线程后，即调用了eglMakeCurrent()之后才能创建，这时OpenGL就能找到上下文数据了。

  所以流程应该是：开启线程-->在线程中初始化EGL-->调用eglMakeCurrent()绑定上下文到线程-->创建shader-->创建program

# shader不能编译，源文件语法错误。这个问题原因有二，一是确实语法错误，修改则矣。另一个又困扰了许久，打印源码，发现是乱
  码。最后发现传参时传的是源码字符串的指针,即指针的指针（const GLchar * * const source），由于glShaderSource()的
  参数就是这个类型，所以就定成了这个类型，然后将这个指针保存到了类里，但是通过*source还是取不到字符串，最后将参数改成了指针，
  在真正调用glShaderSource()时再用取地址符取出字符串地址传给该函数，这时世界和谐了。

# 报错说float类型不向前兼容OpenGL ES 3.0。我明明用的是2.0嘛！经研究错误信息，发现有一处提示小数“1.0f”有问题，
  哦，这下明白了，C语言并没有这种写法，float数后面是不标f的。
